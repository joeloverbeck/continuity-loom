# Observable success criteria

- Pins input ref, resolved SHA, reviewed SHA, literal resolved-SHA three-dot command, commit list, and clean scope.
- Inventories exact Standards and Spec authorities before dispatch and keeps the axes independent.
- Applies the smell baseline as judgement, skips tooling-enforced matters, and maps all four issue requirements including a justified sequence disposition.
- Records reviewer IDs/statuses and truthful unavailable-close cleanup proof.
- Reports visible `## Standards` and `## Spec` blocks with per-axis counts/worst severity and no reranking.
- Avoids implementation-only TDD, evidence-identity, tracker-body, or closeout-line machinery.
