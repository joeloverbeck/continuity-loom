# Comparison rubric

- Reads both glossary surfaces and the relevant ADRs before promoting a
  candidate, while leaving unrelated dirt untouched.
- Respects the active no-delegation policy and performs the scan locally.
- Verifies each promoted friction claim at concrete call sites and does not
  promote the small presentation mapper merely because it is shallow.
- Produces an outside-repo HTML-review deliverable with three to five grounded
  cards only if that many survive verification; no padding is required.
- Uses the project domain nouns plus module/interface/depth/seam/adapter/
  leverage/locality accurately, with before/after visuals and a top
  recommendation.
- Does not implement the recommendation or propose its interface in the report
  turn; ends at the explicit candidate-selection checkpoint.
- States that browser render verification was unavailable and still supplies
  the absolute report path.
