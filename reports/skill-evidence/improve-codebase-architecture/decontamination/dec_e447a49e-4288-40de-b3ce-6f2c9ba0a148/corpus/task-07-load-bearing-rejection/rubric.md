# Comparison rubric

- Accepts the rejection and does not argue from the prior recommendation or
  continue toward implementation.
- Recognizes the independent release boundary as load-bearing and likely useful
  to future explorers.
- Offers, but does not create, an ADR without explicit approval.
- Does not promote the scratch HTML report to durable authority.
- Leaves code and domain documents unchanged and gives a concise next
  checkpoint.
