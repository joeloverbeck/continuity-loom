## Agent Brief

**Category:** bug  
**Summary:** Preserve and display a CAST MEMBER's saved ENTITY selection whenever the record is reopened for editing.

**Current behavior:**  
A CAST MEMBER can be saved with a valid `entity_id`, and that reference remains correct in storage and the API payload. Reopening the CAST MEMBER shows a blank Entity selector because the edit form initializes that control from the wrong field.

**Desired behavior:**  
When an existing CAST MEMBER is opened or reopened, its Entity selector is initialized from the saved `entity_id` and visibly selects the referenced ENTITY. Merely opening the editor must not alter the stored relationship. Existing create, linked-create, validation, and save-failure behavior must remain intact.

**Key interfaces:**

- CAST MEMBER `entity_id` — remains the durable identifier of the linked ENTITY.
- CAST MEMBER Entity selector — reflects the persisted `entity_id` when edit state is initialized.
- Record storage and API payload — remain unchanged.

**Acceptance criteria:**

- [ ] Given an existing CAST MEMBER whose payload contains a valid `entity_id`, opening the editor visibly selects that ENTITY.
- [ ] Creating a linked CAST MEMBER, saving it, and reopening it preserves the same visible Entity selection without requiring the user to reselect it.
- [ ] Saving an otherwise unchanged reopened CAST MEMBER retains the original `entity_id`.
- [ ] Existing create and linked-create initialization, validation, and save-failure retention behavior do not regress.
- [ ] The focused component regression test passes, including access through the selector's existing accessible label.
- [ ] Applicable repository lint, typecheck, test, and build checks pass.

**Out of scope:**

- Stored schema, API payload shape, migrations, export, or provenance changes.
- Prompt compilation, validation-rule, candidate/accepted-prose, or OpenRouter behavior.
- Redesigning ENTITY/CAST MEMBER linking or changing which ENTITY records are eligible for selection.
