# Intake Routing

Load this reference only when the request needs branch-specific intake. `SKILL.md` owns classification and the common question loop.

## Existing design or plan

- Open the cited artifact and selected candidate first; broaden only for dependencies, contradictions, or missing context.
- Read the relevant current principles, glossary, specs, ADRs, implementation, and any named supporting or architecture-vocabulary skill.
- For a coupled proposal, enumerate its load-bearing layers, apply the recommendation to the concrete candidate, and test each layer against current authority, determinism, and failure-mode constraints before choosing the next decision question.
- Check narrow open and relevant closed tracker state when overlap is plausible.
- Treat old reports and reviews as evidence to replay against current source, not as current scope by default.

## Determination or recommendation

- Survey authority indexes, coverage ledgers, current candidates, and relevant open and recently completed work before building the candidate set.
- If a large authority set was requested, read it exhaustively or disclose the bounded strategy.
- Candidate choice is user-owned unless current facts leave only one valid option. Facts may remove candidates; a merely recommended winner remains unratified.
- After candidate, scope, and deliverable depth are settled, stop at the requested recap unless writing, publication, or implementation was explicitly requested.
- Consulting `to-prd` or `to-issues` for house style does not authorize publication.

## Diagnostic or audit

- Let evidence intake dominate: inspect the subject, its authorities, implementation, and relevant tracker state.
- Candidate choice is not required unless the user actually asked to choose among remedies.
- Return a verdict, evidence, rejected or no-op alternatives, recommendation, scope boundary, and freshness.
- If the evidence exposes a conflict that only the owner can settle, ask that intent question rather than choosing by artifact comparison.

## Existing prep

When a matching PRD-ready, issue-ready, or prep artifact exists:

- classify it as current, partly consumed, stale, superseded, or irrelevant;
- name completed or published work that consumed it;
- preserve unconsumed candidates separately; and
- state whether to leave it, refresh it, or replace it.

For a report-cited PRD-ready artifact, continue with [prd-ready-determination-artifact.md](prd-ready-determination-artifact.md).
