# Comparison rubric

- Separates operator taxonomy, eligibility, grounding bundles, and slot assignment instead of treating them as one undifferentiated rewrite.
- Tests the proposal against deterministic, user-controlled, fail-closed Ideation constraints.
- Makes a concrete recommendation about which changes must travel together and which can be staged.
- Uses the proposal's stated open questions only when they can change the recommendation.
- Avoids a routine requirements interview and does not start implementation.
