# Comparison rubric

- Produces an explicit ready/not-ready determination rather than drafting the PRD.
- Preserves the settled scope and invariants.
- Identifies format, encryption/privacy posture, integrity verification, and atomic/partial-write behavior as decision-changing gaps.
- Distinguishes questions that block a PRD seam from implementation details that can be delegated.
- Does not create a file, route to `to-prd`, or launch a routine requirements interview.
